About

grEPI

grEPI is a digital public good for health, a free and open-access software product whose aim is to serve as a living, global, standardized repository (library) of epidemiological parameter estimates by enabling user-friendly online interface for exploring, retrieving and providing data throught web application features as well as API services as machine-readable interface for retrieving desired data programatically.

While the World Health Organization hosts grEPI on its dedicated cloud infrastructure, the product is being designed, developed and maintained by the Epidemiological Parameters Community of Practice under the Collaboratory initiative.

Envisioned as a living resource that continuously evolves through community feedback loops and contributions, grEPI, in its initial proof-of-concept version, brings together epidemiological parameter estimates extracted from the scientific literature by subject matter experts. It provides a central view with standardized vocabulary, such as disease and pathogen context, as well as methodological details and, references to the corresponding data sources. By providing transparent and adjustable views on epidemiological parameter estimates and related data sources, grEPI aims to enable quick identification of relevant epidemiological parameter estimates and support more efficient, reproducible, and consistent epidemiological analyses, strengthening evidence-based decision-making in public health.

Highlights

diamondStandards and quality

To address the main challenge of data heterogeneity and the significant amount of time and effort needed to adjust the data to be able to use it further, grEPI is currently adhering to the WHO International Classification of Diseases(ICD-11), legally cleared WHO Geo Data standard for Health Boundaries for Administrative Level 0 (Countries, Territories, Areas) and is also implementing best practices for classifying and categorizing specific properties of an epidemiological parameter estimate or a literature item.

Using standardized actively maintained vocabulary increases the data quality and enables better understanding of the context.

exploreAccessibility and Usability

Using main features of grEPI, such as browsing and exporting epidemiological parameter estimates and literature in both web application and API, does not bear any licensing cost and does not require creating an application account. As a digital public good for health, access to grEPI is free and open. One only needs to be connected to the Internet to be able to use grEPI in their web browser.

Using grEPI web application interface does not require any specific technical background or skillset. The interface is designed to be user-friendly as possible and to enable smooth user journeys to the desired outcomes.

Learn more arrow_right_alt

lock_open_rightFlexibility and scalability

The design approach of grEPI enables quick adaptation and response to changes in the field with the goal of serving as a global digital public good for health. grEPI is not limited to a disease, pathogen, geography, primary data source or any other specific context. The underlying relational database is designed to support entities and relations rather than hardcoding specific contexts for specific use cases.

Reference data module serves as a core feature of grEPI web application to enable quick configuration of new specific contexts or changes to the existing ones.

User Guide

Getting started with grEPI web application

grEPI is a publicly accessible web application organized into sections for epidemiological parameters, literature, and reference data. It supports browsing, search, filtering, customizable views, and CSV export. For full details on each section and how to use grEPI, see the comprehensive user manual or follow the guide below

Navigate the Application

grEPI web application is organized into three main sections: Epidemiological Parameters, Literature, and Reference data, which are all accessible from the left sidebar.


Interface control:

  • Breadcrumbs: Displayed at the top of each page, it shows your current location within the application (e.g., Home > Reference Data > Diseases).​
  • Diamond-shaped icon: Links to the grEPI API documentation (Swagger UI) - read more on how to use it here.
  • Sign In button: Allows you to sign into or create a Collaboratory account.

Sign In and Access

grEPI can be accessed both with and without logging in. Please note that the full set of reference tables is only available to users who are logged in.

  • Sign-in button: To sign in, click on the sign-in button in the top right corner.
  • Logging-in: You may use your existing WHO Collaboratory account and follow the standard multi-factor authentication (MFA) process.
  • Create an account: If you do not yet have an account, you can create one by following the account registration guidelines provided here.

Epidemiological Parameters - Search function

Epidemiological Parameters view contains the core dataset: epidemiological parameter estimates extracted from the literature. Each row represents a single epidemiological parameter estimate (e.g., an incubation period for Mpox from a specific study).

  • Searching: The search bar above the table performs a full-text multi-term search across all fields currently visible in the grid.
  • To perform a search you can enter multiple keywords separated by spaces, and the table will filter the rows that match all of those terms across any combination of fields.
  • For example, entering "mpox incubation germany" will return rows where the disease is Mpox, the parameter subtype is Incubation period, and the Countries (Population) field includes Germany.

Epidemiological Parameters - Adjust the table

You can adjust the table according to your needs.

  • Sorting: Click a column header to sort the table by that column. The sort direction is indicated by an arrow. ​Only one sort can be active at a time (sorting by a new column replaces the previous sort). The default sort is by Disease (ascending), which groups estimates for the same disease together.
  • Reordering columns: You can reorder columns by clicking on a column header and dragging it to a new position in the table.
  • Pagination: The table displays 20 rows per page by default. The Page size dropdown offers three options: 20, 50, or 100 rows per page. Navigation controls at the bottom allow moving between pages (first, previous, next, last). ​The total number of records is shown (e.g., "1 to 20 of 1000").

Epidemiological Parameters - Filter data

In addition to the search function, you can use multiple column filters to easily locate specific epidemiological parameter estimates.

  • Each column header includes a filter icon. Click it to apply a filter to that column.
  • The filter supports multiple operators: Contains, Does not contain, Equals, Does not equal,Begins with, Ends with, Blank, and Not blank.
  • Multiple column filters can be active at the same time.

Epidemiological Parameters - View all details of the record

The detail view provides a comprehensive breakdown of the selected record, including a high-level summary of the disease, pathogen, and estimate, followed by structured sections covering disease classification and transmission, parameter categorization, and the estimate itself with associated uncertainty.

  • Eye icon: Click the eye icon in the Action column to open a detailed view of a specific epidemiological parameter estimate.
  • Help icon: Hover over the circled question mark next to the field labels for a brief description of what the field means.​
  • Back button: A Back button at the bottom allows you to easily return to the main table.

Epidemiological Parameters - Choose columns to be shown in the table

The full dataset contains over 100 properties of an epidemiological parameter estimate, and properties of the literature and primary data source. The default view shows small subset of properties.

  • Select button: Click the Select button above the table to locate and add additional columns in the table view and in export, or to remove some shown column from the table view.
  • Select All: Click the Select All button if you want to see all available properties in the table view.​
  • Remove All: Click the Remove All button if you want to clear/empty the table view of all properties.

Literature

The Literature section lists the source publications from which parameter estimates have been extracted.

  • Interface controls: Search, column filters, sorting, pagination, select, export and pagination all work the same way as in the Epidemiological Parameters table.
  • Action column: The Action column contains a three-dot menu for each row. Click it to access the History option, which opens a pop-up showing the change history for that entry.​
  • Select button: Click the Select button above the table to locate and add additional columns in the table view and in export, or to remove some shown column from the table view.

Reference Data

Reference Data holds the controlled vocabularies that define the valid values used throughout grEPI.

  • Landing page: The Reference Data landing page displays a card-based overview of all reference data catalogues and mapping tables.
  • Cards: Click any card to open that reference data catalogue.​
  • Interface controls: Search, column filters, sorting, select, export and filter operators work the same way as in the epidemiological parameters section.

Exporting data to your local drive

Each of the three main sections - Epidemiological Parameters, Literature, and Reference Data (individual catalogues) supports data export in CSV format.

  • What gets exported: Columns: The exported CSV includes only the columns that are currently selected via the Select button. By default, only a subset of columns is selected. Rows: By default, the export includes all rows in the table (unfiltered). The export will be filtered in two situations: if you have entered text in the search bar, only rows matching that search will be exported; or if you have selected specific rows using the checkboxes in the left-hand column, only those selected rows will be exported.
  • Configure your export: ​Use the Select button to choose which columns to include. Click Select All button if you want every available field to be included in the exported file. Optionally, apply desired search conditions and filters to include only records of your interests, or use checkboxes in the table to select specific records (entries).
  • Export button: Click the Export button to download the CSV file.
API

Getting started with grEPI API public endpoints

grEPI provides a public, read-only REST API that returns data in JSON format. No authentication is required. The base URL for all requests is https://collaboratory.who.int/grepi/api. String filters are case-insensitive and support partial matching. For full details, see the grEPI API documentation.

Example
Retrieve Measles Estimates with R 
library(httr2)

resp <- request("https://collaboratory.who.int/grepi/api/EpiParameterEstimates") |>
  req_url_query(Disease_Name_Preferred = "Measles") |>
  req_perform()

measles <- resp_body_json(resp, simplifyVector = TRUE)
Example
Retrieve Measles Estimates with Python 
import requests


response = requests.get(
  "https://collaboratory.who.int/grepi/api/EpiParameterEstimates",
  params={"Disease_Name_Preferred": "Measles"}
)

measles = response.json()
